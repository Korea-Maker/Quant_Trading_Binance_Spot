# 문서화 프레임워크 가이드

이 디렉토리는 AI Agent 간 협업을 위한 문서화 프레임워크를 포함합니다.

## 문서 구조

### 1. CONTEXT.md
**목적**: 현재 작업 중인 컨텍스트와 진행 상황

**사용 시점**: 작업 시작 전 필수 확인

**내용:**
- 현재 Phase
- 최근 완료 사항
- 현재 작업 중인 항목
- 다음 작업자를 위한 정보
- 알려진 이슈

**업데이트 주기**: 작업 시작/완료 시

---

### 2. DECISIONS.md
**목적**: 중요한 의사결정과 그 이유 기록

**사용 시점**: 
- 새로운 결정을 내릴 때
- 기존 결정을 이해할 때
- 레거시 코드를 이해할 때

**내용:**
- 각 의사결정의 상황, 고려사항, 결정, 이유
- 우선순위 (Must-have/Should-have/Nice-to-have)
- 영향 범위
- 레거시 코드 정리 가이드

**업데이트 주기**: 중요한 결정 시

---

### 3. ARCHITECTURE.md
**목적**: 시스템 아키텍처 구조와 설계 원칙

**사용 시점**: 
- 새로운 모듈을 추가할 때
- 기존 아키텍처를 이해할 때
- 통합 작업을 할 때

**내용:**
- 시스템 개요
- 모듈 구조
- 프로세스 플로우
- 인터페이스 설계
- 데이터 구조
- 통합 지점

**업데이트 주기**: 아키텍처 변경 시

---

### 4. PRIORITIES.md
**목적**: 기능과 작업의 우선순위 관리

**사용 시점**: 
- 작업 시작 전 우선순위 확인
- 레거시 코드 삭제 전 확인

**내용:**
- Phase별 우선순위 (Must-have/Should-have/Nice-to-have)
- 레거시 코드 관리 가이드
- 다음 작업자를 위한 가이드

**업데이트 주기**: Phase 완료 시, 우선순위 변경 시

---

### 5. AGENT_ROLES.md
**목적**: 각 AI Agent의 역할과 책임 정의

**사용 시점**: 
- 새로운 Agent 생성 시
- Agent 간 작업 분담 확인 시

**내용:**
- 각 Agent의 역할과 책임 범위
- 작업 파일 목록
- 완료 조건
- 작업 순서 및 의존성

**업데이트 주기**: 새로운 Agent 추가 시, 역할 변경 시

---

### 6. AGENT_TASKS.md
**목적**: 각 Agent의 구체적인 작업 내용과 구현 가이드

**사용 시점**: 
- Agent가 작업 시작할 때
- 구체적인 구현 방법이 필요할 때

**내용:**
- 각 Agent의 상세 작업 내용
- 코드 예시 및 구현 가이드
- 완료 기준

**업데이트 주기**: 작업 내용 변경 시

---

## 사용 가이드

### 새로운 작업 시작 시

1. **CONTEXT.md 확인**
   - 현재 Phase 파악
   - 진행 중인 작업 확인
   - 다음 작업 항목 확인

2. **AGENT_ROLES.md 확인**
   - 자신의 역할과 책임 범위 확인
   - 작업 파일 목록 확인
   - 완료 조건 확인

3. **AGENT_TASKS.md 확인**
   - 구체적인 작업 내용 확인
   - 구현 가이드 확인

4. **PRIORITIES.md 확인**
   - 작업의 우선순위 확인
   - Must-have 항목 우선 처리
   - 레거시 코드 주의사항 확인

5. **DECISIONS.md 확인**
   - 관련 의사결정 확인
   - 설계 원칙 이해
   - 레거시 코드 삭제 조건 확인

6. **ARCHITECTURE.md 확인**
   - 모듈 구조 이해
   - 통합 지점 확인
   - 인터페이스 설계 원칙 준수

### 작업 중

1. **의사결정이 필요할 때**
   - DECISIONS.md에 기록
   - 상황, 고려사항, 결정, 이유 명시
   - 우선순위 표시

2. **아키텍처 변경 시**
   - ARCHITECTURE.md 업데이트
   - 변경 이유와 영향 명시

### 작업 완료 시

1. **CONTEXT.md 업데이트**
   - 완료 항목 체크
   - 다음 작업 명시

2. **PRIORITIES.md 업데이트**
   - 완료 항목 ✅ 표시

3. **DECISIONS.md 업데이트** (필요 시)
   - 새로운 의사결정 기록

## 우선순위 가이드

### Must-have (필수)
- **삭제 금지**: 레거시로 표시되어도 삭제 전 마이그레이션 필수
- **우선 처리**: 다른 작업보다 우선
- **테스트 필수**: 완료 전 테스트 통과 필수

### Should-have (권장)
- **조건부 삭제**: 대체 기능 완성 후 삭제 가능
- **순차 처리**: Must-have 완료 후 처리
- **테스트 권장**: 가능한 경우 테스트 추가

### Nice-to-have (선택)
- **자유 삭제**: 필요 없으면 삭제 가능
- **여유 시 처리**: Must-have, Should-have 완료 후
- **테스트 선택**: 선택적

## 레거시 코드 관리

### 레거시 코드 확인 방법

1. **DECISIONS.md 확인**
   - "레거시 코드 정리 가이드" 섹션 확인
   - 삭제 조건 확인

2. **PRIORITIES.md 확인**
   - "레거시 코드 관리" 섹션 확인
   - 우선순위 확인

### 레거시 코드 삭제 체크리스트

- [ ] 대체 기능이 완전히 구현되었는가?
- [ ] 모든 테스트가 통과하는가?
- [ ] 마이그레이션이 완료되었는가?
- [ ] DECISIONS.md에 삭제 이유가 기록되었는가?
- [ ] 다른 모듈에서 참조하지 않는가?

## 예시: 의사결정 기록

```markdown
### [YYYY-MM-DD] 결정 제목

**상황 (Context):**
- 어떤 상황에서 이 결정이 필요했는지

**고려사항 (Considerations):**
1. 대안 1: ...
   - 장점: ...
   - 단점: ...
2. 대안 2: ...
   - 장점: ...
   - 단점: ...

**결정 (Decision):**
- 최종 결정 내용

**이유 (Why):**
- 왜 이 결정을 내렸는지
- 어떤 원칙이나 제약사항이 영향을 미쳤는지

**우선순위:**
- Must-have / Should-have / Nice-to-have

**영향 (Impact):**
- 이 결정이 다른 부분에 미치는 영향

**관련 파일:**
- 파일 경로들
```

## 주의사항

1. **문서는 항상 최신 상태 유지**
   - 작업 완료 시 즉시 업데이트
   - 오래된 정보는 혼란을 야기함

2. **의사결정은 반드시 기록**
   - 나중에 "왜 이렇게 했지?"라는 질문 방지
   - 다른 Agent가 이해할 수 있도록

3. **레거시 코드는 신중하게 삭제**
   - 체크리스트 모두 확인 후 삭제
   - 삭제 이유는 DECISIONS.md에 기록

4. **우선순위는 명확하게**
   - Must-have는 절대 건너뛰지 않음
   - Should-have는 Must-have 완료 후
   - Nice-to-have는 여유 시
