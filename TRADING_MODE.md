# 거래 모드 설정 가이드

## 현재 상태

현재 코드는 **TEST_MODE 설정에 따라 바이낸스 테스트넷 또는 메인넷을 사용**합니다.

## 모드 설정

### 바이낸스 테스트넷 모드 (기본값, 안전)

`.env` 파일에서:
```env
TEST_MODE=True
```

**동작:**
- 바이낸스 테스트넷에서 **실제 주문이 실행됨**
- 테스트넷은 실제 메인넷과 분리된 환경
- 테스트넷 자금만 사용 (메인넷 자금에는 영향 없음)
- 실제 주문 API가 호출되지만 테스트넷에서만 실행됨
- 전략과 시스템을 실제 환경과 동일하게 테스트 가능

**로그 예시:**
```
[바이낸스 테스트넷] 매수 주문 실행: 100.0 USDT
```

**테스트넷 API 키 발급:**
1. https://testnet.binance.vision/ 접속
2. 테스트넷 계정 생성
3. 테스트넷 API 키 발급
4. `.env` 파일에 테스트넷 API 키 설정

### 바이낸스 메인넷 모드 (주의 필요)

`.env` 파일에서:
```env
TEST_MODE=False
```

**동작:**
- 바이낸스 메인넷에서 **실제 주문이 실행됨**
- 실제 자금이 사용됨
- 실제 손익이 발생함
- **주의해서 사용하세요!**

**로그 예시:**
```
[바이낸스 메인넷] 매수 주문 실행: 100.0 USDT
```

## 현재 설정 확인

현재 설정을 확인하려면:

```bash
python -c "from src.config.settings import TEST_MODE; print(f'TEST_MODE: {TEST_MODE} (테스트넷)' if TEST_MODE else f'TEST_MODE: {TEST_MODE} (메인넷)')"
```

## 주의사항

1. **기본값은 테스트넷 모드**입니다 (`TEST_MODE=True`)
2. 메인넷에서 실제 거래를 하려면 **명시적으로 `TEST_MODE=False`로 설정**해야 합니다
3. 메인넷 사용 전에 **반드시 테스트넷에서 충분히 테스트**하세요
4. `.env` 파일을 백업하고 안전하게 보관하세요
5. **테스트넷과 메인넷의 API 키는 다릅니다** - 각각 별도로 발급받아야 합니다

## 테스트넷 사용 방법

1. **테스트넷 계정 생성**
   - https://testnet.binance.vision/ 접속
   - 테스트넷 계정 생성 및 로그인

2. **테스트넷 API 키 발급**
   - 테스트넷 계정에서 API 키 생성
   - API Key와 Secret Key 복사

3. **`.env` 파일 설정**
   ```env
   TEST_MODE=True
   BINANCE_API_KEY=테스트넷_API_KEY
   BINANCE_API_SECRET=테스트넷_SECRET_KEY
   ```

4. **실행 및 확인**
   - 시스템 실행 시 테스트넷에 연결됨
   - 실제 주문이 테스트넷에서 실행됨
   - 메인넷 계정에는 영향 없음

## 테스트넷의 장점

- 실제 주문 API를 사용하므로 실제 환경과 동일
- 메인넷 자금에 영향 없음
- 실제 주문 실행 흐름을 정확히 테스트 가능
- 네트워크 지연, 주문 체결 등 실제 환경과 유사
- 전략 검증 후 메인넷으로 안전하게 전환 가능
